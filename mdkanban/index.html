mdkanbanについて

mdkanbanは、Markdownファイル（.md）をブラウザにドラッグ&ドロップするだけでカンバンボードとして表示し、カードのドラッグ&ドロップで列の並び替えや列間移動もそのまま行える無料ツールです。Obsidian Kanbanプラグインで作成したボードファイルをそのまま開け、見慣れた列ごとのレイアウトでタスクを俯瞰しながら状態を更新できます。

並び替えた結果は元のMarkdown構造を保ったまま、Chrome系ブラウザでは元ファイルへ直接保存（File System Access API）、その他のブラウザではダウンロードまたはクリップボードへのコピーで持ち出せます。すべての処理はお使いのブラウザ内で完結し、ファイルがサーバーに送信されることはありません。情報セキュリティに敏感な業務でのMarkdown閲覧・並び替えにもご利用いただけます。

使い方

  1. このページを開く（インストール・登録は不要です）
  2. Markdownファイル（.md）をページ内にドラッグ&ドロップする、または「ファイルを開く」ボタンから選択する
  3. 自動でカンバンボードとして表示されます（## 見出しが列、- 箇条書きがカードに変換されます）
  4. カードをドラッグして別の列にドロップ、または同じ列内で上下に並び替えできます。カード上の鉛筆アイコンでタイトルをインライン編集、ゴミ箱アイコンで削除、各列下部の「+ カード追加」で新規作成できます。詳細モーダルでは本文（Markdown）とサブタスクをまとめて編集できます
  5. カードを行（プロジェクト別・担当者別など）でもグループ化したい場合は、frontmatter に lanes: キー（YAML 配列）でレーン名を宣言してから、カード文字列に #lane/<名前> を追記します（例：frontmatter に lanes: [バックエンド, フロントエンド]、カードに - [ ] API設計 #lane/バックエンド）。スイムレーン表示に切り替わり、レーンの追加・削除・並び替え・名前変更はレーンヘッダから直接操作でき、その結果は lanes: へ自動的に書き戻されます
  6. Chrome系で File System Access API 経由でファイルを開いた場合は、編集／DnD／追加／削除のたびに 800ms デバウンスで元ファイルへ自動保存されます（ヘッダーで「💾 保存中…」「✓ 保存済み」を確認できます）。その他のブラウザでは「ダウンロード」「コピー」ボタンで持ち出せ、LocalStorageにも自動保存され再訪時に復元できます
  7. テーマ（ライト／ダーク）と表示密度（コンパクト／詳細）はヘッダーから切替できます。設定はブラウザに保存されます

主な機能

  • Obsidian Kanban互換: kanban-plugin: basic frontmatter付きファイルもそのまま読み込み可能
  • カードのドラッグ&ドロップ: 列間移動・同列内の並び替えに対応。HTML5標準のDnD APIを使用しPC操作に最適化
  • カードの編集・追加・削除: カード上の鉛筆/ゴミ箱アイコンと「+ カード追加」ボタンで、タイトル・本文・サブタスクを直接編集可能。すべての変更が再シリアライズされます
  • Markdownへの書き戻し: Chrome系はFile System Access APIで元ファイルに直接上書き保存（変更から800msデバウンスで自動保存）、その他のブラウザはダウンロード／クリップボードコピー（双方ともLocalStorageへも自動反映）
  • タグ＆期限バッジ: #tag はバッジ表示、@YYYY-MM-DD は今日基準で赤（期限切れ）／黄（当日）／灰（未来）に自動色分け
  • サブタスク進捗: ネストした箇条書きを 2/5 形式の進捗バッジで表示
  • スイムレーン（行グルーピング）: frontmatter に lanes: キー（YAML 配列）でレーン名を宣言すると、行で分かれた 2D グリッド表示に切り替わります。lanes: の順序がそのままレーン表示順になり、カード側は #lane/<名前> でレーン所属を指定します。lanes: に未列挙の #lane/X や lane 未指定のカードは「未分類」レーンへ集約されます。レーンヘッダから追加・削除・並び替え・名前変更が可能で、行間のDnDや行の折りたたみ（状態はLocalStorageに保存）にも対応。Obsidian Kanban 公式はスイムレーン未対応のため、同じファイルを Obsidian で開いても lanes: は未知 frontmatter として、#lane/X はタグバッジとして表示されるだけでデータは壊れません
  • カード詳細モーダル: 段落・コードブロック・リンクをフルレンダリング
  • タグフィルタ: バッジクリックで該当タグのカードのみ絞り込み（フィルタ中もDnDで列移動可）
  • 復元機能: 最後に開いたファイル内容をブラウザに保存し、再訪問時に復元提案
  • ダークモード対応: ヘッダーのトグルから切替、設定は永続化
  • XSS対策: Markdown内の <script> や javascript: URLはサニタイズして無害化

2つのカンバンmdの差分を確認したいときは、同じKIRICAB Toolsのテキスト比較ツールと組み合わせるとレビューがスムーズです。

よくある質問

どんなMarkdown形式に対応していますか？

Obsidian Kanbanプラグインの ## 見出し＝列、- [ ] タスク＝カード形式に対応しています。frontmatter（---で囲まれたYAML部分）もパースして無視します。kanban-plugin: basic の有無は問いません。

アップロードしたファイルはサーバーに送信されますか？

いいえ。すべての処理はお使いのブラウザ内で完結します。ファイル内容が外部のサーバーに送信されることはありません。社内Wikiや個人ノートのMarkdownでも安心してご利用いただけます。

カードの並び替えや編集はできますか？

はい。カードをドラッグして別の列にドロップしたり、同じ列内で上下に並び替えたりできるほか、カードタイトル・本文・サブタスクの編集、カードの追加・削除にも対応しています（カード上の鉛筆/ゴミ箱アイコン、各列下の「+ カード追加」ボタン、モーダルの編集ボタン）。Chrome系ブラウザで File System Access API 経由でファイルを開いた場合は変更が 800ms デバウンスで自動的に元ファイルへ書き戻され（保存中は「💾 保存中…」、完了で「✓ 保存済み」をヘッダーに表示）、その他のブラウザでは「ダウンロード」「コピー」ボタンで持ち出せます。LocalStorageにも自動保存されるため、リロード時の復元バナーから編集後の状態を呼び戻せます。

2つのカンバンmdの差分を比較したいときは？

同じKIRICAB Toolsのテキスト比較ツール（diff）をご利用ください。並び替え前後のMarkdownを左右に貼り付ければ、列とカードの移動を色分けでレビューできます。

タグや期限の色はカスタマイズできますか？

期限バッジは今日基準で自動色分け（赤＝期限切れ／黄＝当日／灰＝未来）しており、カスタマイズには対応していません。タグ自体の配色を考えるときはカラーパレット生成ツールもご活用いただけます。

対応していない記法はありますか？

担当者記法 @username（日付以外の @文字列）や複数H1ボードなどは現バージョン非対応です。期限記法 @YYYY-MM-DD のみ特別扱いし、それ以外の @文字列 は通常テキストとして表示します。

読み込めるファイルサイズの上限は？

1ファイルあたり5MBまでです。これを超えるファイルはエラー表示となります。1000カード規模のmdまでは快適に表示できるよう設計しています。

スイムレーン（行グルーピング）はどう使いますか？

frontmatter に lanes: キー（YAML 配列）でレーン名を宣言し、カード本文に #lane/<レーン名> を書くと、その行に振り分けられます。lanes: の出現順がそのまま表示順になります。lanes: に未列挙の #lane/X や lane 未指定のカードは末尾の「未分類」レーンへ集約されます（カード側の #lane/X はバッジに出ません）。レーンの追加・削除・並び替え・名前変更は画面上のレーンヘッダから操作でき、lanes: の値は自動的に Markdown へ書き戻されます。lanes: キーが無いファイルではスイムレーン機能はオフになり、#lane/X は通常のタグバッジとして表示されます（厳密モード）。

レーン（行）の追加・削除・並び替えはできますか？

はい。スイムレーン表示の末尾にある「+ レーン追加」ボタンで新規レーンを追加でき、各レーンヘッダの ▲ / ▼ ボタンまたはヘッダのドラッグで並び替え、✎ ボタンで名前変更、🗑 ボタンで削除できます（削除時は所属カードが「未分類」レーンに集約されます）。「未分類」レーンは末尾固定で名前変更・削除はできません。すべてのレーン操作は frontmatter の lanes: 配列に書き戻され、所属カードのタイトル中の #lane/<旧名> も新名へ一括置換されます。lanes: が空になった場合は lanes: キーごと削除され、従来モード（単一行）に自動で戻ります。

Obsidian Kanban で同じファイルを開いたとき lane 関連はどう見えますか？

Obsidian Kanban 公式はスイムレーン未対応のため、frontmatter の lanes: キーは未知 frontmatter として無視され、カード中の #lane/X は単なるタグバッジとして表示されるだけです。ファイル内容は破壊されないため、mdkanban 側で lane を 2D 表示に展開する「緩い互換」設計のもと、Obsidian と mdkanban の両方で同じファイルを安心して行き来できます。
