# mdkanban — Markdownカンバンビューワー

## 目的・概要
Markdownファイルを読み込むだけで、見出しと箇条書きをそのままカンバンボードとして可視化し、カードのドラッグ&ドロップで列の並び替えや列間移動を行えるブラウザ完結型ツール。Obsidian Kanbanプラグイン互換のMarkdownを「ログイン不要・全部ローカル・サーバー送信なし」で開けることを最大の価値とする。タスク管理SaaS（Trello/Notion等）の代替を目指すのではなく、「手元のMarkdownをカンバン形式で眺めたい／状態だけ並び替えたい」という限定ニーズに特化する。並び替え結果はMarkdownへ再シリアライズし、Chrome系ブラウザでは元ファイルへ直接上書き保存（File System Access API）、その他のブラウザではダウンロード／クリップボードコピーで持ち出せる（PC優先・モバイルは閲覧のみ）。

## 対象ユーザーとユースケース

### 想定ユーザー（ペルソナ）
- **P1: Obsidianユーザー** — Obsidian Kanbanプラグインで作ったボードを、Obsidianを開けない環境（共有PC・スマホブラウザ）で閲覧したい
- **P2: テキスト派エンジニア／ライター** — タスクをMarkdownでGit管理しており、レビューや進捗確認時にビジュアルなカンバン表示で俯瞰したい
- **P3: 情報セキュリティに敏感なユーザー** — タスク内容を外部SaaSにアップロードしたくないが、生のMarkdownを読むのは見づらいと感じている

### 主なユースケース
1. 手元の `kanban.md`（Obsidian Kanban形式）をドラッグ&ドロップして、出先のブラウザで内容を確認する
2. GitHub上のIssue一覧をMarkdown化したファイルを開き、列ごとの分布を一目で把握する
3. プロジェクトのREADMEに含めた進捗マークダウンを、関係者にURLとファイル両方で共有しブラウザで読んでもらう

## 既存ツールとの関係
- **重複・類似ツール有無**: 無し（既存8ツールはパスワード／絵文字／Diff／配色／QR／印鑑／あみだくじ／写真フィルターで、ドキュメント系は未提供）
- **mdeditorとの関係**: 完全に独立。mdeditorはまだ未リリース／リリースされない可能性があるため、依存・相互参照は行わない
- **クロスリンク候補**:
  - `diff/`（テキスト比較）— 「カンバンmdの2バージョンを比較したいときはDiffへ」という導線をFAQ／本文に設置
  - `colorpallet/`（カラーパレット）— カードのタグ色設定の参考として軽く触れる程度（FAQで1箇所）

## Markdown→Kanban変換ルール（核心仕様）

### 基本構造（Obsidian Kanban Plugin互換）
```
---
kanban-plugin: basic
---

# ボードタイトル（任意・H1）

## 列名1（H2）

- [ ] 未完了タスク
- [x] 完了タスク

## 列名2（H2）

- 箇条書きのみのカード（チェックボックスなし）
```

### 変換ルール詳細

| Markdown要素 | Kanban上の扱い |
|---|---|
| YAML frontmatter | パースする。`kanban-plugin: basic` の有無は問わない。frontmatter内の `title` キーがあればボードタイトルの**初期値**として採用する（H1が無いときのフォールバック）。**`lanes:` キー（YAML 配列）はスイムレーン定義のホワイトリスト**として扱う（後述「スイムレーン」参照）。それ以外のキーは無視 |
| `# 見出し`（H1） | ボードタイトル（ヘッダーに表示）。複数あっても最初の1つのみ採用。**frontmatter.title と両方が存在する場合はH1を優先**し、再シリアライズ時もH1テキストが保たれる |
| `## 見出し`（H2） | **列**。出現順に左から並べる |
| `### 以下`（H3〜H6） | カード本文の見出しとしてレンダリング（列やカードの区切りにはしない） |
| `- [ ] テキスト` | 未完了カード（タイトル=テキスト） |
| `- [x] テキスト` または `- [X] テキスト` | 完了カード。**取り消し線＋薄色（不透明度50%程度）で表示**。列は変えない |
| `- テキスト` | チェックボックスなしカード（未完了扱い） |
| 箇条書きのネスト（カードの子要素） | サブタスクとして扱う。表示モードに応じて：<br>・コンパクト表示：「完了/総数」のバッジのみ（例：`2/5`）<br>・詳細表示：チェックボックス付きで列挙（読み込み専用、操作不可） |
| カード直下の段落・コードブロック・リンク・画像 | カード詳細ビュー（カードクリックで開くモーダル）でMarkdownレンダリングして表示 |
| カード本文中の `#tag` | カード下部にバッジ表示。クリックで全カードに対するタグフィルタを発動 |
| カード本文中の `@YYYY-MM-DD` | 期限バッジ。今日基準で：<br>・**期限切れ（過去）：赤**<br>・**当日：黄**<br>・**未来：灰**<br>形式: `@2026-05-01` |
| カード本文中の `@文字列`（日付以外） | MVPでは**通常テキストとして扱う**（担当者機能は次フェーズ） |
| カード本文中の `#lane/<名前>` | **スイムレーン（行グルーピング）指定**。**frontmatter に `lanes:` キーが存在する場合のみ lane 専用記法として解釈**される（厳密モード）。`lanes:` 配列に列挙された名前と一致した場合：そのレーンに割り当て、タグバッジに出さず `displayTitle` からも除去。`lanes:` に**未列挙**の名前を指定した場合：「未分類」レーンに合流するが、`#lane/X` 自体はタグバッジに出さず `displayTitle` からも除去する（lane 専用記法であることに変わりはないため）。1カードに複数あっても**最初の1つのみ採用**、`#lane/foo/bar` は `foo` のみ採用。lane名の文字種は通常タグと同じ（空白不可）。**`lanes:` キーが frontmatter に無いファイルではスイムレーン機能は一切作動せず、`#lane/X` は通常のタグバッジとして表示される**（`displayTitle` 除去も行わない） |

### 寛容パース方針
- frontmatterが無いMarkdownでも動作する
- H2が1つも無い場合は「全カードを `カード` という単一列に入れる」フォールバックを行う
- H2より前にある箇条書きは「列に属さない」ため、`未分類` 列にまとめる
- インデントはスペース2/4、タブのいずれも許容

### スイムレーン（行グルーピング）
**Obsidian Kanban 公式はスイムレーン未対応**（Issue #237 / #650 が長年未解決）であり、コミュニティの代替策も Obsidian プラグイン依存でブラウザ完結ツールには持ち込めない。そのため mdkanban では独自拡張として **frontmatter の `lanes:` キー（YAML 配列）でレーンを宣言（ホワイトリスト）し、カード側は `#lane/<名前>` タグでレーン所属を指定する**ハイブリッド方式を採用する。Obsidian Kanban で同じファイルを開いた場合、`lanes:` キーは未知の frontmatter として無視され、`#lane/...` は単なるタグバッジとして表示されるだけで、ファイル内容を破壊することはない（緩い互換）。

#### 採用記法
**スイムレーンを使うファイル（frontmatter で `lanes:` を宣言）:**
```markdown
---
kanban-plugin: basic
lanes:
  - バックエンド
  - フロントエンド
---

## TODO
- [ ] API設計 #lane/バックエンド
- [ ] 画面実装 #lane/フロントエンド
- [ ] スプリント計画                ← lane未指定（「未分類」レーン）
- [ ] 検証用スクリプト #lane/QA      ← lanes: に未列挙（「未分類」レーンに合流。#lane/QA はタグバッジに出ない）

## Doing
- [ ] DBマイグレーション #lane/バックエンド #design
```

**スイムレーンを使わないファイル（`lanes:` キーが無い）:**
```markdown
---
kanban-plugin: basic
---

## TODO
- [ ] 画面実装 #lane/フロントエンド   ← #lane/フロントエンド はそのままタグバッジに表示される
```

#### 動作モード（厳密モード）
mdkanban は frontmatter に `lanes:` キーが**ある／ないか**でモードを切り替える。

| `lanes:` キー | モード | `#lane/X` の扱い | スイムレーン UI | `.has-swimlanes` クラス |
|---|---|---|---|---|
| 無い | 従来モード（単一行カンバン） | **通常のタグバッジとして表示**。`displayTitle` から除去しない | 出ない | 付かない |
| ある（空配列含む） | スイムレーンモード | **lane 専用記法として解釈**。`displayTitle` から除去しタグバッジに出さない | 出る（`lanes:` の順序通り） | 付く |

#### `lanes:` モードでのカード割り当てルール
- カードに `#lane/X` が**有り**かつ X が `lanes:` 配列に**列挙されている** → 該当レーンに配置
- カードに `#lane/X` が**有り**かつ X が `lanes:` 配列に**未列挙** → 「未分類」レーンに合流（ただし `#lane/X` はタグバッジに出さず `displayTitle` から除去する）
- カードに `#lane/X` が**無い** → 「未分類」レーンに合流
- 「未分類」レーンは `lanes:` の末尾に固定で表示する。カードが1件も該当しない場合は非表示にしてよい
- 専用の「未定義」レーンは設けない

#### 制約
- lane名の文字種は通常タグと同じ `[A-Za-z0-9_\-぀-ヿ㐀-鿿]+`（空白不可）。`lanes:` 配列要素も同じ文字種制約に従う
- 1カードに複数の `#lane/X` がある場合は**最初の1つのみ採用**
- `#lane/foo/bar`（多階層）は MVP では先頭セグメント `foo` のみ採用
- `#lane/`（名前なし）は無効 → 「未分類」扱い（タグバッジに出さず `displayTitle` から除去）
- `lanes:` 配列に重複した名前が含まれる場合は最初の出現のみ有効とし、以降の重複は読み込み時に黙って除去する
- **`lanes:` に列挙されていない `#lane/新規` がカードに付いていても自動追記しない**。新しいレーンを増やしたいときは UI のレーン追加操作を使う

#### 表示
- 横軸が列（H2）、縦軸がレーン（`lanes:` 配列の順序＋末尾「未分類」）の 2D グリッド
- レーンヘッダはレーン名・折りたたみボタン・**並び替えハンドル**・**レーン名編集ボタン（実装する場合）**・**削除ボタン**を持つ
- レーンヘッダクリック／キーボード（Enter / Space）で行を折りたたみ／展開可能。状態は `localStorage.mdkanban.collapsedLanes` に JSON 配列で保存
- モバイル幅でも横スクロールで閲覧でき、レーンラベル列は左端に sticky で残す

#### レーン管理 UI（MVP）
スイムレーンの追加・削除・並び替えは UI で完結する（`lanes:` を手書きで編集する必要はない）。詳細仕様は F8 を参照。

#### Markdown シリアライズ規約（スイムレーン関連）
- `lanes:` キーの順序がレーンの表示順序と一致するように再シリアライズする
- 既存の他 frontmatter キー（`kanban-plugin: basic` 等）の並び・値・空行を破壊しない（`lanes:` 以外のキーは入力時の順序を保つ）
- `lanes:` を新規生成する場合（=「スイムレーンを有効化」）の挿入位置：
  - frontmatter が**既に存在する場合** → 既存 frontmatter の**末尾**に追記する
  - frontmatter が**存在しない場合** → `---` ブロックを新規に作成し、`lanes:` のみを含む frontmatter としてファイル先頭に挿入する
- `lanes:` 配列が**空になった場合**は **`lanes:` キーごと frontmatter から削除する**（＝従来モードに戻る）。`lanes: []` のままは残さない。これにより「全レーン削除＝スイムレーン無効化」がユーザーの直感と一致する
- `lanes:` 削除の結果 frontmatter が空（`---` のみ）になった場合は frontmatter ブロックごと削除する
- YAML 配列のシリアライズ形式は `- 名前` ブロックスタイルで統一する（フロー形式 `[a, b]` は使わない）。レーン名に YAML 予約文字が含まれる場合のみ必要なクォートを行う

### サンプルmd → 変換結果のペア例

**入力 (`example.md`)**
```markdown
---
kanban-plugin: basic
---

# 春のリリース計画

## TODO

- [ ] ロゴ刷新 #design @2026-05-10
  - [ ] 案出し
  - [ ] レビュー会
- [ ] OGP画像生成 #seo

## Doing

- [ ] ドキュメント整備 @2026-04-25
  詳細はこちらを参照: [社内Wiki](https://example.com)

## Done

- [x] 仕様書作成 #planning
- [x] 関係者ヒアリング
```

**変換結果（UI上の表現）**
- ヘッダー: 「春のリリース計画」
- 列: TODO（2件）／Doing（1件）／Done（2件・取り消し線表示）
- TODOの1枚目「ロゴ刷新」: タグ `#design`、期限 `2026-05-10`（灰）、サブタスク `0/2` バッジ
- Doingの「ドキュメント整備」: 期限 `2026-04-25`（赤・期限切れ）、カードクリックでWikiリンク付き本文がモーダル表示
- Doneの2枚: 取り消し線＋50%不透明度

## 機能要件

### 必須機能（MVP）

#### F1. ファイル読み込み
- F1-1: `<input type="file" accept=".md,.markdown,text/markdown">` でファイル選択
- F1-2: ページ全体（または明示的な大きなドロップゾーン）への**ドラッグ&ドロップ**で `.md` を読み込む
- F1-3: ドラッグオーバー中はドロップゾーンがハイライトされ、「ここに離すと読み込み」というメッセージを表示
- F1-4: Chromiumブラウザで `window.showOpenFilePicker` が利用可能な場合は「ファイルを開く（推奨）」ボタンも併設（オプション）
- F1-5: 読み込んだファイル名をヘッダーに表示

#### F2. パース＆カンバン表示
- F2-1: 上記「Markdown→Kanban変換ルール」に基づき、列とカードを描画
- F2-2: 列は横並び、列内は縦に積まれたカード
- F2-3: カードはクリックで詳細モーダル（フルMarkdownレンダリング）を開ける
- F2-4: タグバッジクリックで該当タグを持つカードのみ表示するフィルタ
- F2-5: 期限バッジは色分けして視認性を高める

#### F3. 表示設定
- F3-1: テーマ切替（**ライト／ダーク**）。トグルボタン1つで切替
- F3-2: カード表示密度切替（**コンパクト／詳細**）。コンパクトはタイトル＋バッジのみ、詳細は本文冒頭1〜2行のプレビュー含む
- F3-3: タグフィルタ解除ボタン

#### F4. 永続化（LocalStorage）
- F4-1: 最後に開いたMarkdownの**内容**を保存し、次回訪問時に「前回開いたファイルを復元しますか？」と提示。承諾で復元、ファイル名も併せて保存
- F4-2: テーマ・カード密度設定を保存
- F4-3: タグフィルタ状態・カードモーダルの開閉状態は保存しない（セッション内のみ）

#### F5. オンボーディング／空状態
- F5-1: 初回訪問・ファイル未読込時はドロップゾーン中央に「Markdownファイルをドラッグ＆ドロップ、またはここをクリックして選択」と表示
- F5-2: 「サンプルを表示」ボタンを設置し、上記サンプルmdをロードしたデモ表示が可能
- F5-3: 初回オンボーディングとして「対応Markdown形式」の簡易ガイド（`## が列、- がカード` の最小例）を空状態画面に常時表示

#### F6. レスポンシブ＆タッチ対応
- F6-1: PC幅では列を横並び、画面幅を超えたら横スクロール
- F6-2: モバイル幅（〜599px）でも横並びを維持し、横スクロールで列を移動できる
- F6-3: タッチ操作で列の横スクロールが慣性スクロール（`-webkit-overflow-scrolling: touch` 等の標準的な配慮）でスムーズに動作する
- F6-4: カードのモーダルはモバイル幅では画面ほぼ全面を占めるダイアログとして表示

#### F7. カードのドラッグ&ドロップ並び替えとMarkdown書き戻し
- F7-1: HTML5 Drag and Drop API 標準のみ使用し、外部DnDライブラリ（SortableJS等）は導入しない（ライセンス・サイズ管理の単純化のため）
- F7-2: 各カードに `draggable="true"` を付与し、列間移動および同一列内の上下並び替えに対応する
- F7-3: ドラッグ中はドラッグ元カードを半透明（不透明度0.4程度）化し、ドロップ予定位置に水平インジケーター線を表示する
- F7-4: ドロップ完了時に `state.board` を更新→Markdown再シリアライズ→`localStorage` の `mdkanban.lastContent` を新内容で上書きし、未保存マーカー（●）をツールバーに表示する
- F7-5: 再シリアライズは frontmatter / H1 / H2 / チェック状態 / `#tag` / `@日付` / サブタスクを完全保証し、bodyParts段落・コードブロックの空白構造はベストエフォートで保持する（論理構造は不変）。H1テキストは frontmatter.title より優先採用され、入力時のH1がそのまま再シリアライズ後にも維持される
- F7-6: タグフィルタ適用中もDnDによる列間移動は許可し、フィルタ状態を維持する。非表示カードに対する位置挿入は不可（visibleカード間にのみインジケーターが出る）
- F7-7: Chromium系（File System Access API対応）でファイルを開いた／FSA経由のドロップでハンドルが取得できた場合、「💾 保存」ボタンが表示され、押下時に元ファイルへ上書き保存する
- F7-8: 全ブラウザ共通で「⬇ ダウンロード」「📋 コピー」ボタンが利用可能。コピーは `navigator.clipboard.writeText` を優先し、失敗時は `document.execCommand('copy')` フォールバック
- F7-9: 未保存変更がある状態でページ離脱しようとした場合、`beforeunload` で確認ダイアログを発火させる
- F7-10: スクリーンリーダー向けにライブリージョン（`aria-live="polite"`）でDnD完了をアナウンスする（例:「カード○○を△△列の◎番目に移動しました」）
- F7-11: モバイル幅でのDnD・FSAは公式サポート対象外（タッチDnDの実装複雑度とSortableJS非導入の方針を優先）。閲覧自体はモバイルでも引き続き可能
- F7-12: 通常の `<input type="file">` 経由で開いたファイルでは保存ボタンは非表示（FSAハンドルが無いため上書き不可）。リロード後はFSAハンドルが失効するため、再度ファイルピッカーで開き直すことで保存ボタンが復活する

#### F8. スイムレーン（行グルーピング）
- F8-1: **モード判定**: frontmatter に `lanes:` キーが**有る**md はスイムレーンモードで描画し、`lanes:` 配列の順序通りにレーンを並べる。`lanes:` キーが**無い**md は従来モード（単一行カンバン）で描画する。`.has-swimlanes` クラスはスイムレーンモード時のみ付与する
- F8-2: **カード割り当て**: スイムレーンモード時、カードの `#lane/X` が `lanes:` に列挙されていればそのレーンへ配置。未列挙または `#lane/` 未指定のカードは「未分類」レーンへ合流する。「未分類」レーンは `lanes:` の末尾に固定で表示し、該当カードが0件のときは非表示にしてよい
- F8-3: **複数 `#lane/X` の扱い**: 1カードに複数の `#lane/X` がある場合は最初の1つのみ採用。`#lane/foo/bar` は `foo` のみ採用する
- F8-4: **`#lane/X` の表示**: スイムレーンモードでは `#lane/X` をタグバッジに表示せず、`displayTitle` からも除去する（`lanes:` 未列挙の名前であっても同様）。**従来モード（`lanes:` キーが無い）では `#lane/X` は通常のタグバッジとして表示し、`displayTitle` から除去しない**
- F8-5: **カード DnD**: スイムレーン間でカードを DnD して移動でき、`card.lane` を更新して再シリアライズ時にカードタイトル中の `#lane/<旧>` を `#lane/<新>` に書き換える。「未分類」レーンへ移動した場合は `#lane/X` をカードタイトルから除去する
- F8-6: **折りたたみ**: レーンヘッダクリック／キーボード（Enter / Space）で行を折りたたみ／展開でき、状態は `localStorage.mdkanban.collapsedLanes` に JSON 配列で保存される
- F8-7: **モバイル**: モバイル幅でも横スクロールでスイムレーンを閲覧でき、レーンラベル列は左端に sticky で残す
- F8-8: **a11y アナウンス**: `aria-live="polite"` リージョンで「<title> を <lane名> レーンの <列名> 列の N 番目に移動しました」形式（従来モードのときは「<列名> 列の N 番目」）

##### F8-A: レーン追加 UI
- F8-A-1: スイムレーンエリアの末尾（または専用の「+ レーン追加」ボタン）からレーンを追加できる。ボタン押下でレーン名入力欄（インライン `<input>` または小さなプロンプト）を表示し、確定でレーンを `lanes:` 配列の末尾に追加する
- F8-A-2: `lanes:` キーが frontmatter に**無い**ファイルでも初回追加で `lanes:` キーを生成する（=UI からスイムレーンを「有効化」できる）。frontmatter 自体が無い場合は frontmatter ごと新設する
- F8-A-3: 同名レーンが既に `lanes:` 配列にある場合は追加を拒否し、入力欄に「すでに存在するレーン名です」とインラインバリデーションを表示する
- F8-A-4: lane 名は `[A-Za-z0-9_\-぀-ヿ㐀-鿿]+`（空白不可）の文字種制約に従う。違反時はインラインバリデーションを表示する
- F8-A-5: 追加確定後、`state.board.lanes` 更新 → 再シリアライズ → `localStorage.mdkanban.lastContent` 更新 → 自動保存トリガー → 再描画

##### F8-B: レーン削除 UI
- F8-B-1: 各レーンヘッダに削除ボタン（🗑 または `×`）を配置する。「未分類」レーンには削除ボタンを表示しない
- F8-B-2: 削除ボタン押下で確認ダイアログを表示する。文言は「レーン「<lane名>」を削除します。所属する N 件のカードは「未分類」レーンに移動されます。よろしいですか？」とする
- F8-B-3: 確定で `lanes:` 配列から該当レーンを除去し、そのレーンに属していたカード全ての `#lane/<lane名>` をカードタイトルから除去する（カード自体は削除しない・「未分類」レーンへ集約される）
- F8-B-4: 削除の結果 `lanes:` が空配列になった場合は `lanes:` キーごと frontmatter から削除し、従来モード（単一行カンバン）に自動的に切り替わる
- F8-B-5: 削除完了は `aria-live="polite"` で「レーン「<lane名>」を削除しました（N 件のカードを未分類に移動）」とアナウンスする

##### F8-C: レーン並び替え UI
- F8-C-1: レーンヘッダの DnD（ドラッグハンドル）または上下矢印ボタンで上下に並び替えできる
- F8-C-2: 「未分類」レーンは末尾固定で並び替え不可（DnD 対象から除外、矢印ボタンも表示しない）
- F8-C-3: 並び替え確定で `lanes:` 配列の順序を更新し、再シリアライズ → 自動保存 → 再描画する
- F8-C-4: 並び替え操作中はドラッグ中レーンヘッダを半透明化し、ドロップ予定位置に水平インジケーターを表示する

##### F8-D: レーン名変更 UI（MVPに含む）
- F8-D-1: レーンヘッダのレーン名部分をダブルクリック、または専用の編集ボタン押下でインライン編集モードに入る
- F8-D-2: `Enter` で確定、`Esc` でキャンセル、`blur` でも確定。同じ文字種制約・重複禁止バリデーションを適用する
- F8-D-3: 確定時、`lanes:` 配列内の該当エントリを新名に置換すると同時に、**そのレーンに属する全カードのタイトル中の `#lane/<旧名>` を `#lane/<新名>` に一括置換**する
- F8-D-4: 「未分類」レーンは名前変更不可（編集 UI を表示しない）
- F8-D-5: 確定後、再シリアライズ → 自動保存 → 再描画する

#### F9. カード編集（タイトル・本文・サブタスク）
- F9-1: 各カードの**右上に編集ボタン（鉛筆アイコン `✏️` または `✎`）を表示**する。表示タイミングは**カード hover／focus 時のみ表示**を推奨（コンパクト表示密度でのレイアウト崩れ回避と視覚ノイズ軽減のため）。タッチデバイスはhoverが効かないので、`(hover: none)` メディア環境では常時表示にフォールバックする
- F9-2: カード上の編集ボタン押下で**インライン編集モード**に入り、タイトル領域が `<textarea>` に置き換わる。`Enter` で確定、`Shift+Enter` で改行（タイトル文字列内に改行を許容するかは仕様外。MVPでは改行は禁止し、`Enter`は常に確定とする）、`Esc` でキャンセル。`blur` でも確定する
- F9-3: カード詳細モーダル（クリックで開く既存モーダル）に「**編集**」ボタンを追加し、閲覧モードと編集モードを**トグル**で切替可能とする。編集モードに切り替えるとモーダル内のタイトル・本文・サブタスクが入力可能領域に変化する
- F9-4: モーダル編集モードでは以下を編集できる:
  - **タイトル**: 1行 `<input type="text">`（タグ `#x`・期限 `@2026-05-10`・lane `#lane/X` を含めた**生のタイトル文字列を直接編集**する）
  - **本文（bodyParts）**: 複数行 `<textarea>` で **Markdown を直接編集**（段落・コードブロック・リンク・画像参照等）
  - **サブタスク**: チェック状態（`[ ]`/`[x]`）と各サブタスクのテキストを**行単位で編集**できる。行追加（「+ サブタスク追加」ボタン）と行削除に対応。並び替えはMVPスコープ外
- F9-5: 編集確定（インライン: Enter/blur、モーダル: 「保存」ボタン）で `state.board` のカード情報を更新し、Markdown を再シリアライズして `localStorage.mdkanban.lastContent` を更新、再描画する。タイトル中の `#lane/X` を編集した場合は `card.lane` も再パースして反映する（タグ・期限も同様に再抽出）
- F9-6: モーダル編集モードでは「キャンセル」ボタンと `Esc` で変更を破棄して閲覧モードに戻る。インライン編集も `Esc` で破棄
- F9-7: 編集ボタン・編集モード内の各操作は `<button>` でセマンティックに実装し、Tab でフォーカス可能、Enter / Space で操作可能とする。`aria-label` を付与する（例: `カード「○○」を編集`、`カード「○○」を削除`）
- F9-8: カードがインライン編集モードまたはモーダル編集モードにある間、当該カード（および編集UIに乗っている要素）の `draggable` を `false` に切り替えて DnD を無効化する。編集モードを抜けたタイミングで `draggable=true` に戻す

#### F10. カード追加・削除
- F10-1: 各**列の最下部**（既存カード群の末尾）に「**+ カード追加**」ボタンを表示する。スイムレーン使用時は**各セル（lane × 列）の最下部**にそれぞれ表示する
- F10-2: クリックで「タイトル空・本文空・サブタスク無し」の新規カードがその位置（列／lane の末尾）に追加され、**即座にインライン編集モードに入りタイトル `<textarea>` にフォーカス**する
- F10-3: 新規カードがタイトル空のまま `blur` または `Esc` でキャンセルされた場合は**そのカードを破棄**する（state.board に残さない）。1文字でも入力されて確定された場合は通常カードとして登録される
- F10-4: 各カードに「**削除**」操作を提供する。配置案:
  - **カードのhover/focus時に右上の編集ボタンの隣にゴミ箱アイコン（🗑 or `×`）を表示**（編集ボタンと同じ表示ルール）
  - **モーダル閲覧モードのフッターにも「削除」ボタン**を併設（ボリュームのあるカードは中身を確認してから削除したいニーズに対応）
- F10-5: 削除操作時は**確認ダイアログ（`window.confirm` 相当）を表示**し、確定で削除、キャンセルで何もしない（データロスト防止のため確認は必須）。確認文言は「カード「<タイトル>」を削除します。よろしいですか？」とする
- F10-6: 追加／削除後は `state.board` を更新し、Markdown 再シリアライズ → `localStorage.mdkanban.lastContent` 更新 → 自動保存トリガー（F11）→ 再描画を行う
- F10-7: 削除完了は `aria-live="polite"` リージョンに「カード「<タイトル>」を削除しました」とアナウンスする。追加完了は新規カードの編集状態に入る性質上、明示的アナウンスは不要

#### F11. FSA自動保存
- F11-1: FSA ハンドルがある（FSA 経由でファイルを開いた）場合、**変更（DnD・編集確定・カード追加・カード削除）から 800ms デバウンス**で自動的にファイルへ書き戻す（連続変更は最後の1回に集約）
- F11-2: 自動保存中はツールバー上の保存状態インジケーターに「**💾 保存中…**」を表示し、書き込み完了後「**✓ 保存済み**」を**約1.5秒**表示してフェードアウトする。アイドル時はインジケーターを非表示またはニュートラル表示にする
- F11-3: 自動保存に失敗した場合（writable 失敗・I/O エラー等）は「**⚠ 保存失敗**」エラーをトーストで表示し、`dirty` フラグ（`●` マーカー）を残したまま、**手動の「💾 保存」ボタンで再試行**できる状態を保つ
- F11-4: FSA ハンドルが**無い**（`<input type="file">` 経由・サンプル・LocalStorage 復元のみ）場合は**自動保存しない**。従来通りの「⬇ ダウンロード」「📋 コピー」ボタンによる手動持ち出しに委ねる
- F11-5: 「💾 保存」ボタンは引き続き表示し、**即時手動保存トリガー**としても機能する（デバウンスを待たずに即座に書き込む）
- F11-6: ページ離脱時の `beforeunload` 確認ダイアログは「**dirty かつ（自動保存中 or 直近の自動保存が失敗している）**」場合のみ発火する。自動保存が完了して保存済み状態であれば `beforeunload` ダイアログは出さない（ユーザーストレス軽減）
- F11-7: FSA 権限が剥がれた・writable がエラーを返す等で自動保存が**継続不能**と判定された場合、エラー表示を出し、自動保存を一時停止して「⬇ ダウンロード」「📋 コピー」ボタンを再露出する（FSA ハンドル無し状態と同等の挙動にフォールバック）
- F11-8: 自動保存の対象とする「変更」は state.board の構造的変化に限る（テーマ／密度切替・タグフィルタ操作・モーダル開閉は対象外）

## 非機能要件
- ブラウザ完結 / 日本語UI / 商用利用可ライセンスのみ
- レスポンシブ対応（モバイル375px〜）
- アクセシビリティ：キーボード操作（Tab／Enter／Escでモーダル閉じる）、aria属性、4.5:1以上のコントラスト
- SEO（OGP共通画像、FAQ／HowTo／WebApplication／BreadcrumbListのJSON-LD）
- パフォーマンス：1000カード規模のmdをパース＆描画して操作がもたつかないこと
- セキュリティ：読み込んだMarkdown内のHTML／script／`javascript:` URLは無害化（XSS防止）。画像のsrcは `http(s):` と `data:` のみ許可

## 入出力の定義

### 入力
- **形式**: `.md` / `.markdown` テキストファイル（UTF-8想定）
- **上限**: 1ファイル5MBまで（超過時はエラー表示）
- **バリデーション**:
  - 拡張子チェック（`.md` / `.markdown` 以外は警告のうえ続行可）
  - 文字コード判定不可時はUTF-8で読み込みを試行、失敗時はエラー
  - 中身が空文字列／H2が無くて箇条書きも無い場合は「カンバン化できる要素がありません」と表示

### 出力
- **画面表示のみ**（MVPではダウンロード／コピー／共有なし）
- LocalStorageへ最後の内容と表示設定のみ保存

## UI・UX要件

### 画面構成
1. **ヘッダー**: ツール名「mdkanban」／読み込み済みファイル名／テーマトグル／密度トグル／「ファイルを開く」ボタン／「サンプル」ボタン／**保存状態インジケーター（💾 保存中… / ✓ 保存済み / ⚠ 保存失敗 / ● 未保存）** ／「💾 保存」「⬇ ダウンロード」「📋 コピー」ボタン
2. **メインエリア**:
   - ファイル未読込時 → 大きなドロップゾーン＋オンボーディング案内
   - 読み込み後 → カンバンボード（列の横並び）。各列／（スイムレーン使用時）各セルの**最下部に「+ カード追加」ボタン**
3. **タグフィルタバー**（タグが存在する場合のみ表示）: アクティブなタグと「クリア」ボタン
4. **フッター**: 既存ツール群と同じスタイルのリンク群＋著作権表示

### カード上の操作ボタン配置（F9・F10対応）
- カードの**右上に小さな操作ボタン群**を配置する。左から「✏️ 編集」「🗑 削除」の順
- **表示タイミング**: PC（hover 可能環境）では**カード hover／focus 時のみ表示**してビジュアルノイズを抑える。タッチ環境（`@media (hover: none)`）では**常時表示**にフォールバック
- ボタンの大きさはタップ領域として最低 24×24px を確保し、コンパクト密度でもタイトル文字とぶつからないよう右上に absolute 配置する
- ボタンはセマンティックな `<button>` で実装し、`aria-label` に対象カードのタイトルを含める

### モーダルのレイアウト
- **閲覧モード（既定）**: 現状維持（タイトル H2／本文の Markdown レンダリング／サブタスクのチェックリスト表示／タグ・期限バッジ）。モーダル右上に「✏️ 編集」、フッターに「🗑 削除」「閉じる」ボタン
- **編集モード（「✏️ 編集」押下後）**: 同じモーダル内で以下を縦に配置
  1. **タイトル入力**: `<input type="text">`（タグ・期限・lane を含む生のタイトル文字列）
  2. **本文編集**: 複数行 `<textarea>`（Markdown を直接編集、等幅フォント推奨）
  3. **サブタスク編集**: 行ごとに「チェックボックス + テキスト入力 + 行削除ボタン」を縦に並べ、最下部に「+ サブタスク追加」ボタン
  4. **アクション**: 「保存」「キャンセル」ボタン（フッター）
- 編集モード中は閲覧モードのレンダリングは隠す。`Esc` キーは編集モード破棄として優先動作し、モーダル閉じは「キャンセル」または「閉じる」ボタンに集約する

### 操作フロー
1. ユーザーがページを開く → 空状態（前回ファイルがあれば復元提案バナー）
2. ファイルをドロップ／選択／サンプルクリック → 即座にパースしてカンバン描画
3. カードをクリック → 詳細モーダルで全Markdownを整形表示
4. タグバッジをクリック → そのタグのカードのみに絞り込み、上部にフィルタ状態表示
5. テーマ／密度トグル → 即座に反映、設定はLocalStorageに保存
6. 別のファイルを開く → 既存表示を破棄して新ファイルを描画
7. カードの編集ボタン押下 → インライン編集（タイトル）／モーダル編集（タイトル＋本文＋サブタスク）→ 確定で再シリアライズ＆自動保存
8. 「+ カード追加」押下 → 空カードが追加され即編集モード → 入力後 Enter で確定／空のまま blur で破棄
9. カード削除ボタン押下 → 確認ダイアログ → 確定で削除＆自動保存
10. FSA で開いた状態での編集／DnD → 800ms デバウンス後にファイルへ自動書き戻し（ヘッダーのインジケーターで状態確認）

### フィードバック
- **成功**: パース完了時、列数とカード数を `3列・12枚を表示中` のように小さくトースト or ヘッダーに表示
- **エラー**: ファイル形式不正・サイズ超過・パース失敗時、画面中央にエラーパネル＋復旧ボタン（再選択）
- **警告**: H2が無くて単一列フォールバックを使った場合、ボード上部に `## 見出しが見つからなかったため、すべてのカードを1列にまとめました` と非破壊的に通知
- **保存状態**: ヘッダーのインジケーターで `● 未保存` / `💾 保存中…` / `✓ 保存済み` / `⚠ 保存失敗` を視覚的に区別し、状態変化はテキスト＋色＋アイコンで複合的に表現する（色だけに頼らない）
- **自動保存失敗**: 画面右下にトーストで「⚠ 保存に失敗しました。再試行してください」と表示し、手動「💾 保存」ボタンで再試行できる旨を案内
- **削除確認**: `window.confirm` 相当のダイアログで「カード「<タイトル>」を削除します。よろしいですか？」を表示

### エッジケース
- 空ファイル: 「ファイルが空です」エラー
- frontmatterのみ: 「カンバン化できる要素が見つかりません」エラー
- H2はあるが箇条書きが0件: 空の列を表示（エラーにしない）
- 巨大ファイル（5MB超）: 「ファイルサイズが上限を超えています」エラー
- バイナリファイル: 「テキストファイルとして読み込めませんでした」エラー
- カード本文が極端に長い: コンパクト表示でも崩れないよう `text-overflow: ellipsis` でクリップ。詳細はモーダルで全文表示
- ネストが深いサブタスク: 表示モーダルでは2階層まで描画（それ以上はフラット化を検討）

## 画面遷移・状態管理

### 状態
- `unloaded`（未読込・初回）
- `restorable`（LocalStorageに前回ファイルあり、復元提案バナー表示中）
- `loading`（パース中、サイズ大の場合の一瞬）
- `loaded`（カンバン表示中）
- `error`（読み込み／パースエラー）

### 遷移
- `unloaded` → ファイル選択 → `loading` → 成功で `loaded`、失敗で `error`
- `restorable` → 「復元する」で `loaded`、「閉じる」で `unloaded`
- `error` → ファイル再選択で `loading` へ
- `loaded` → 別ファイル選択で `loading` へ

### `seo-content` セクション内容

#### 使い方（HowToと対応・3〜5ステップ）
1. mdkanbanのページを開く
2. Markdownファイル（`.md`）をドラッグ&ドロップする、または「ファイルを開く」ボタンから選択する
3. 自動でカンバンボードとして表示される（`## 見出し`が列、`-` 箇条書きがカード）
4. カードをクリックすると詳細を表示。タグや期限が含まれていれば自動でバッジ化される
5. テーマ・表示密度はヘッダーから切替できる（設定はブラウザに保存）

#### よくある質問（FAQ項目案）
1. **どんなMarkdown形式に対応していますか？**
   → Obsidian Kanbanプラグインの `## 見出し`＝列、`- [ ] タスク`＝カード形式に対応。frontmatterもパースして無視します。
2. **アップロードしたファイルはサーバーに送信されますか？**
   → いいえ。すべての処理はお使いのブラウザ内で完結します。ファイルは外部に送信されません。
3. **カードの並び替えや編集はできますか？**
   → カードのDnDによる列間移動・同列内並び替えに加え、**カードタイトル・本文・サブタスクの編集**、**カードの追加・削除**にも対応しています。Chrome系で File System Access API 経由でファイルを開いた場合は変更が **800ms デバウンスで自動的に元ファイルへ書き戻され**、その他のブラウザ／開き方ではDL/コピーで持ち出せ、LocalStorage へも自動反映されます。
4. **2つのカンバンmdの差分を比較したいときは？**
   → テキスト比較ツール「[diff](../diff/index.html)」をご利用ください。同じkiricab.github.ioで提供しています。
5. **タグや期限の色はカスタマイズできますか？**
   → 期限は今日基準で自動色分け（赤＝期限切れ・黄＝当日・灰＝未来）です。タグの配色を考える時は「[カラーパレット生成](../colorpallet/index.html)」もご活用ください。
6. **対応していない記法はありますか？**
   → 担当者記法 `@username` や複数H1ボードなどは現バージョン非対応です。期限記法 `@YYYY-MM-DD` のみ特別扱いします。
7. **読み込めるファイルサイズの上限は？**
   → 1ファイルあたり5MBまでです。これを超えるファイルはエラー表示となります。
8. **スイムレーン（行グルーピング）はどう使いますか？**
   → frontmatter に `lanes:` キー（YAML 配列）でレーン名を宣言し、カード本文に `#lane/<レーン名>` を書くと、その行に振り分けられます。レーンの追加・削除・並び替え・名前変更は画面上のレーンヘッダから操作でき、`lanes:` の値は自動的に Markdown へ書き戻されます。`lanes:` キーが無いファイルではスイムレーン機能はオフになり、`#lane/X` は通常のタグバッジとして表示されます。
